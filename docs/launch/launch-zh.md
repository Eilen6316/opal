**标题:** 开源了一个"给 Word/Excel 开 PR"的层:AI 只提议、人来审、外科手术式写回

**正文:**

做了个开源项目 OtterPatch(Apache-2.0,纯开源、不商业化),解决一个我一直不放心的事:
**让 AI 直接改文件太危险,但逐字自己改又太累。**

它的做法是把代码圈的 PR 流程搬到文档上:

- **AI 只提议,不动文件。** 圈选一段文字/一片单元格,说"把这段排成公文格式"——Agent 产出一份
  结构化的 ChangeSet,还能通过取数工具通读全文(而不是从截断的上下文里瞎猜)。
- **系统先自检,再给你看。** 每条改动的锚点(引文/单元格)必须真实、唯一地落得下去;公式要能
  重算;流程图的边不能悬空——不合格的提案直接打回让模型自己修,你看到的已经是过了机器审的。
- **审阅像 Word 修订。** 工作区里红删绿增行内展示,逐条接受/拒绝,四种视图随时切换
  (原文/修订/清样/改后);Excel 是真实网格上的前后回放,拒绝一条不误伤同格的另一条。
- **写回是外科手术。** 只动被改的 XML 部件:真实 531KB docx 实测 31 个部件 30 个字节不变
  (模型整文重写会动 11 个)。Word 改动落成**原生修订**,拿给领导继续审都行。

中文场景做了些别人没有的:**GB/T 9704 公文排版打法手册**(标题小标宋、正文三号仿宋、
一、/(一)/1. 层级体系自动规范)、财务表勾稽检查、双栏论文版式。

BYOK 支持 8 家模型(DeepSeek/GLM/Kimi/豆包/MiniMax/Claude/Gemini/OpenAI),
也可以作为 MCP server 接进 Claude Code 等任何 Agent。
